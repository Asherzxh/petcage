该源码分为三部分，硬件部分和软件前端，软件后端
硬件部分采用esp32作为主控，Ardunio框架下开发。
一、 核心控制部分
ESP32-S3 开发板
功能： 系统主控中枢， 负责数据处理、 逻辑判断、 硬件控制与联网通信。技术实现细节： 依托 240MHz 双核处理器， 实时接收传感器数据并完成滤波校准； 通过板载
WiFi 建立网络连接， 采用 HTTP 协议实现数据上传云端与指令接收； 采用循环调度机制， 同
步监听远程指令与执行本地自控， 实现双模式无冲突运行。
二、 传感采集部分
1. SHT3X 温湿度传感器
功能： 实时采集箱内温度、 湿度数据。
技术实现细节： 通过 I2C 接口与主控板通信， 将模拟温湿度信号转为数字信号上传， 为环境
温湿度调控提供精准数据。
2. SGP30 气体传感器
功能： 监测箱内二氧化碳、 有机气体浓度， 检测空气质量。
技术实现细节： 采用 I2C 协议传输数据， 将气体浓度信号转换为数值信号， 判断是否启动通
风操作。
3. HX711 重力传感器（2 个）
功能： 分别检测食盆、 水槽内食物与饮用水余量。
技术实现细节： 将重力信号转为电信号， 经模数转换后上传主控板， 通过预设阈值判断是否
启动补给操作。
三、 执行控制部分
1. SG90 舵机（2 个）
功能： 控制食盆、 水槽开合， 完成自动喂食、 加水。
技术实现细节： 接收主控板 PWM 信号， 旋转指定角度， 精准控制投喂与加水量。
2. 5V 直流水泵（2 个）
功能： 分别实现自动加水、 定时清理粪便。
技术实现细节： 由 MOS 管控制通断， 接收指令后启动； 清理水泵每小时定时自动运行， 将粪
便抽至收集槽。
3. 12V 风扇、 加热片
功能： 风扇通风换气， 加热片调控箱内温度。
技术实现细节： 通过 MOS 管实现弱电控强电， 当环境数据超标时自动启动， 达标后自动停止。
4. MOS 管开关（3 个）
功能： 驱动大功率器件， 保护主控板， 保障电路安全。
技术实现细节： 接收主控板电平信号控制电路通断， 避免大电流损坏核心控制板。四、 结构与供电部分
1. 外壳及定制部件
功能： 防护、 美观、 防水， 适配硬件安装。
技术实现细节： 亚克力+雪弗板拼接， 热熔胶固定、 密封胶泥防水； 3D 打印食盆水槽， 贴合
传感器布局， 保障余量检测精准。
2. DC 电源
功能： 为全系统提供稳定供电。
技术实现细节： 匹配各部件额定电压， 持续稳定供电， 保障系统不间断运行。
五、 配套外设部分
1. 智慧宠物项圈
功能： 采集宠物心率、 体温、 血氧健康数据。
技术实现细节： 锂电池供电， 安全且可重复充电。 内置生理传感器， 通过无线联网将数据
传输至主控板， 实现箱体与项圈数据联动。
2. 监控摄像头
功能： 远程实时监控箱内状态。
技术实现细节： 使用内网穿透， 将摄像头内网穿透外网使之接入系统， 将视频流上传云端，

软件
二． 软件技术栈
1. 前端：
项目采用 SPA 架构， 页面集中在 src/pages， 路由覆盖登录、 状态、 控制、 报告、 我
的等核心入口。
service 层按业务域拆分： CageStatus、 PetMetrics、 FeedingHistory、 Alert、 Command、
Report 等。
HTTP 层统一 baseURL=/api、 10s 超时和 JSON 请求头， 并把 Pinia 中的 token 注入
Authorization。
移动端设计按 390px 宽度组织， 底部导航固定， 操作反馈通过 Toast 及时提示。
2. 后端
启动类为 org.example.petcage.PetCageApplication， 服务端口配置为 8080
后端按 Controller、 Service、 ServiceImpl、 Mapper、 Pojo、 DTO、 VO、 Result、 Exception
分层组织
业务对象覆盖 Cage、 CageEnv、 CageControl、 Pet、 PetHealth、 FeedRecord、 WaterRecord、
WarningRecord、 PetWeeklyReport
配置中开启下划线转驼峰映射， Mapper XML 位于 classpath:mapper/*.xml， 并使用
JWT 拦截器保护接口

整体功能实现逻辑
系统采用数据采集-处理分析-执行响应-云端同步闭环运行： 传感器实时采集各类数据
上传主控板， 主控板比对阈值后自动驱动执行器件； 所有数据同步上传云端， 用户可下发远
程指令； 定时任务与自主调控结合， 实现全自动+远程手动双模式智能养护。
三． 软件方面：
前端通过购买云服务器， 搭建服务器存放网站， 后端通过编写数据库， 接受硬件数据，
并上传到服务器， 同时用户指令通过后端发给硬件。
